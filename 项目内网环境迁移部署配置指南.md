# 项目内网环境迁移部署配置指南

## 概述

本文档描述将 AI 数据分析工作台项目迁移到内网环境时，需要修改的各项配置信息。
**注意：**
- 所有配置项都必须根据内网环境进行修改，不能保持默认值。
- 数据库连接信息（如主机名、端口、用户名、密码）必须根据内网数据库服务器的实际情况进行调整。

---

## 一、后端服务配置

### 文件位置
`server/.env`

### 配置项说明

| 配置项 | 当前值 | 内网环境说明 |
|--------|--------|--------------|
| `PORT` | `3001` | 后端服务监听端口，根据内网规划调整 |
| `DB_HOST` | `localhost` | **必填修改**：内网数据库服务器地址 |
| `DB_PORT` | `3306` | 数据库端口，MySQL 默认 `3306` |
| `DB_USER` | `root` | **必填修改**：内网数据库用户名 |
| `DB_PASSWORD` | `123456` | **必填修改**：内网数据库密码 |
| `DB_NAME` | `ai_workbench` | 数据库名称，保持不变或根据内网规范调整 |
| `JWT_SECRET` | 默认值 | **必填修改**：更换为安全的密钥，建议至少32位随机字符串 |
| `JWT_EXPIRES_IN` | `7d` | Token过期时间，根据安全要求调整 |

### 配置示例

```env
PORT=3001
DB_HOST=192.168.1.100
DB_PORT=3306
DB_USER=internal_user
DB_PASSWORD=your_secure_password_here
DB_NAME=ai_workbench
JWT_SECRET=your_256_bit_secure_secret_key_must_be_long
JWT_EXPIRES_IN=7d
```

## 二、前端生产环境变量

### 文件位置
创建 `.env.production`（参考 `.env.production.example`）

### 配置项说明

| 配置项 | 说明 |
|--------|------|
| `VITE_BANK_AGENT_BASE_URL` | 银行报告Agent后端服务的内网地址 |

### 配置示例

```env
# 银行报告Agent后端地址（内网部署）
VITE_BANK_AGENT_BASE_URL=http://192.168.1.101:8001
```

## 三、前端运行时配置

前端支持运行时动态注入配置，通过 `window.runtime_config` 对象传递：

```javascript
// 在 HTML 页面中注入（通常由部署脚本完成）
<script>
window.runtime_config = {
  VITE_BANK_AGENT_BASE_URL: 'http://192.168.1.101:8001'
}
</script>
```

**优先级**：运行时配置 > 环境变量 > 默认值

## 七、部署步骤建议

1. **准备数据库**：在目标内网数据库服务器创建数据库和用户
2. **配置后端**：修改 `server/.env` 文件，更新数据库连接信息
3. **配置前端**：创建 `.env.production` 文件，配置服务地址
4. **构建前端**：执行 `pnpm build` 生成生产环境代码
5. **启动服务**：启动后端服务和前端静态资源服务
6. **验证测试**：登录系统验证各项功能正常

---

## 八、安全注意事项

1. **密钥管理**：`JWT_SECRET` 不应存储在代码仓库中，应通过环境变量注入
2. **数据库密码**：避免使用弱密码，建议使用至少12位包含大小写字母、数字和特殊字符的密码
3. **网络隔离**：确保数据库仅允许内网特定IP访问
4. **HTTPS**：生产环境建议配置HTTPS证书
5. **日志审计**：开启适当的日志记录，便于安全审计
