# AI数据分析工作台使用手册

## 1. 项目定位

AI数据分析工作台用于把业务分析需求转化为可执行的分析流程。用户可以填写报告需求，配置数据源，维护指标 SQL，调用模型生成分析报告初稿，并对结果进行复制、二次编辑或沉淀为模板。

适用角色：

- 数据分析师：维护指标、生成分析结论、整理报告。
- 业务人员：输入经营分析需求，快速获得报告初稿。
- 开发人员：扩展数据源、模型适配、报告生成链路和桌面端能力。

## 2. 本地运行

### 2.1 环境准备

- Node.js：22.x
- 包管理器：pnpm

首次安装：

```bash
pnpm install
```

### 2.2 启动 Web 版本

```bash
pnpm dev
```

启动后按终端提示访问本地地址，通常是 `http://localhost:18181` 或 Vite 输出的端口。

### 2.3 启动桌面版本

```bash
pnpm dev:desktop
```

### 2.4 构建

构建 Web、扩展和核心包：

```bash
pnpm build
```

构建桌面应用：

```bash
pnpm build:desktop
```

## 3. 基础使用流程

1. 打开应用后进入“AI数据分析工作台”。
2. 在顶部导航选择“智能报告”。
3. 在报告需求输入区填写业务需求，例如：

```text
生成一份 2025 年第四季度某支行经营分析报告，重点分析存款、贷款、客户增长和风险变化。
```

4. 选择“数据分析报告Agent”。
5. 点击“优化/生成”按钮，系统会整理报告目标、核心指标和输出结构。
6. 打开“数据源管理”，添加数据库连接信息并测试连接。
7. 在指标列表中维护或补充 SQL，将指标绑定到对应数据源。
8. 执行报告生成，查看报告初稿。
9. 根据业务事实检查数据、结论和措辞，必要时继续修改需求或指标 SQL 后重新生成。

## 4. 数据源管理

数据源配置用于保存数据库连接信息。常见字段包括：

- 数据库类型
- 主机地址
- 端口
- 数据库名
- 用户名
- 密码

建议：

- 本地开发使用测试数据库或脱敏数据。
- 不要把真实数据库密码写入代码或提交到 Git。
- 生产环境连接信息应通过环境变量、部署平台密钥或安全配置中心管理。

## 5. 指标 SQL 管理

报告模式支持把报告指标与 SQL 绑定。每个指标通常包含：

- 指标名称
- 指标说明
- 绑定数据源
- SQL 查询语句
- 输出字段说明

SQL 编写建议：

- 查询范围要与报告时间、机构、产品等条件一致。
- 字段命名尽量清晰，便于模型理解。
- 避免直接查询敏感明细；优先提供聚合结果。
- 复杂指标可拆分为多个子指标，减少单条 SQL 的复杂度。

## 6. 模型配置

进入“模型管理”后可配置模型服务。通常需要填写：

- 服务商或兼容接口类型
- API 地址
- API Key
- 默认模型名
- 显示名称

注意：

- API Key 属于敏感信息，只保存在本地或安全环境中。
- 如果连接失败，优先检查 API 地址、模型名称、网络代理和密钥权限。

## 7. 数据导入导出

“数据管理”用于备份或迁移应用配置。导出内容可能包含历史记录、模型配置、自定义模板、收藏和部分用户设置。

使用建议：

- 重要配置变更前先导出备份。
- 导入会覆盖现有数据，请确认备份文件来源可信。
- 不要把包含密钥或客户数据的导出文件提交到仓库。

## 8. 多人开发流程

推荐分支模型：

```text
main        稳定分支，只合并已验证代码
develop     日常集成分支
feature/*   功能开发分支
fix/*       缺陷修复分支
```

常用流程：

```bash
git checkout develop
git pull
git checkout -b feature/report-datasource
pnpm install
pnpm dev
```

提交前检查：

```bash
pnpm lint
pnpm test
```

提交示例：

```bash
git add .
git commit -m "feat(report): add datasource management"
git push origin feature/report-datasource
```

## 9. Git 上传范围

应该提交：

- `packages` 下的源码
- `api`、`scripts`、`tests`、`docs` 等项目代码和文档
- `package.json`、`pnpm-lock.yaml`、配置模板和构建配置

不应该提交：

- `.env.local`
- `node_modules`
- `dist`、`build`、桌面安装包等构建产物
- `scratch` 临时目录
- 日志、缓存、测试报告
- 私钥、证书、真实生产密钥

## 10. 常见问题

### 依赖安装失败

确认 Node.js 是 22.x，并使用 pnpm：

```bash
node -v
pnpm -v
```

### 页面标题或文案没有更新

重新启动 dev server，必要时清理浏览器缓存。

### 桌面端构建失败

先单独构建 Web：

```bash
pnpm build:web
```

再执行：

```bash
pnpm build:desktop
```

### Git 不小心显示大量临时文件

先确认 `.gitignore` 是否生效：

```bash
git status --short
```

如果临时文件已经被添加到暂存区，应取消暂存后再提交必要代码。
